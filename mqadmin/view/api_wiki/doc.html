所有静态资源均在这个目录下
1、MYSQL尽量不要联表或者子查询
2、尽量不要在循环里写SQL QUERY
3、代码结构，尽量什么类型的代码放在哪个目录正面
比如配置就放在config，静态文件就放在www下。
日志也最好统一写在同一个文件夹。
不紧好维护，也方便日后做扩展
4、URL的东西，域名尽量不要写死到DB中，最好是写相对路径，PHP根据项目名，动态计算域名
5、能封装的东西，尽量封装成公共类。一些静态值，最好不要写死，定义成 常量，方便修改
6、MYSQL建表的时候，都要加上注释信息
7、在开发的过程中，要有一个记事本，记录如果上线修改的东西，如：配置信息、MYSQL修改、修改了哪些文件
8、所有的APP接口开发，都走驱动测试模式，也就是：统一有一个接口配置文件，包括：请求参数、参数类型、返回等等。能通过程序调用API文档，自动化请求并测试
9、代码中，尽量给核心代码写上注释，方便日后查看
10、代码风格不做强求。能按照JAVA的方式最好，如：驼峰式。尽量:(人类)能看懂你写的代码，少用如：三目运算这些不易阅读的方式 。
11、实时性要求不高的需求，和产品沟通下，能扔缓存里的都扔进去。。
12、调试API接口时，要会使用抓包工具，如：Wireshark，同时要有记日志的习惯，方便调错.
13、每天下班关电脑前，必须把当天所有开发代码，提交到SVN服务器。
